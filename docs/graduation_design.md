# 毕业设计说明文档

## 题目

非接触式老年人居家监测系统设计与实现

## 研究背景与意义

随着老龄化程度加深，独居或半独居老年人的居家安全问题越来越突出。传统照护方式依赖人工巡查，实时性和连续性不足。本设计基于 ESP32 构建低成本居家监测终端，通过环境、人体活动、气体、压力、振动和 SOS 等信息实现异常状态识别，并通过声光、电器联动完成本地提醒，为后续扩展远程通知提供基础。

## 总体方案

系统由感知层、控制层、执行层和显示交互层组成。

感知层包括 DHT22、BH1750、MQ135、MQ7、HC-SR501、SW420、FSR402 和 SOS 按键；控制层采用 ESP32-S3-N16R8 开发板；执行层包括蜂鸣器、LED 灯、继电器风扇和 SG90 舵机；显示交互层采用 SSD1306 OLED 与串口日志。系统还利用 BH1750 与 PIR 的组合实现起夜辅助：当环境较暗且检测到人体活动时，自动打开 LED 灯并延时关闭。

## 硬件设计

ESP32 通过 I2C 总线连接 OLED 和 BH1750，通过 ADC 采集 MQ135、MQ7 与 FSR402 的模拟量，通过数字 IO 采集 PIR、SW420 和 SOS 按键状态。继电器、蜂鸣器、LED 和舵机由 ESP32 GPIO 输出控制。

供电方面，ESP32 与低压传感器使用 3.3V，风扇、继电器和舵机使用 5V，所有模块共地。MQ 系列传感器和舵机电流较大，不建议直接由 ESP32 3.3V 引脚供电。

## 软件设计

软件采用 Arduino Framework 编写，并使用 PlatformIO 管理依赖。程序采用模块化配置方式：各传感器和执行器的引脚、阈值放在 `include/devices/` 下的独立配置文件中，主程序只保留采集、判断、显示和联动流程，便于后续更换模块或调整阈值。

程序主要包含：

1. 设备初始化：配置 GPIO、I2C、OLED、BH1750、DHT22、舵机和 ADC。
2. 数据采集：按固定周期读取各传感器数据。
3. 阈值判断：根据温湿度、气体、压力、振动、人体活动和 SOS 状态生成告警，同时根据暗环境和 PIR 活动判断老人起夜状态。
4. 联动控制：根据告警状态控制蜂鸣器、继电器风扇、LED 和舵机。
5. 信息显示：OLED 展示核心数据，串口输出调试日志。

## 关键算法

系统采用多源阈值融合的规则判断方法。气体、温湿度和压力采用阈值比较；PIR 用于更新最近活动时间，当超过设定时间没有检测到活动时产生提醒；在光照低于起夜阈值且 PIR 检测到人体活动时，系统判定老人可能起夜，自动点亮 LED 并保持一段时间；SOS 按键使用软件消抖，避免机械抖动导致误触发。

## 测试方案

- 上电测试：OLED 显示启动信息，串口输出 `ESP32 elder monitor started.`。
- 温湿度测试：用手靠近 DHT22，观察温湿度变化。
- 光照测试：遮挡 BH1750，观察 Lux 降低，LED 自动点亮。
- 起夜测试：遮挡 BH1750 并在 PIR 前移动，观察 OLED 显示 `NIGHT MOVE`，LED 自动点亮并保持一段时间。
- 气体测试：靠近 MQ135/MQ7 使用安全气源模拟，观察风扇继电器和告警变化。
- 人体测试：人在 PIR 前移动，观察 PIR 状态变为 Y。
- 振动测试：轻敲 SW420 模块，观察振动告警。
- 压力测试：按压 FSR402，观察压力数值升高。
- SOS 测试：按下按键，蜂鸣器响，舵机转动。

## 可扩展方向

- 增加 WiFi 和 MQTT/HTTP 上传，实现手机端或网页端远程查看。
- 增加 NTP 时间同步和离线数据缓存。
- 使用多次采样平均或滑动窗口滤波降低误报。
- 增加外壳结构设计与 PCB 优化，提高系统可靠性。
